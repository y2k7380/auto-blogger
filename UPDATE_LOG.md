# Auto-Blogger Pro 업데이트 로그 (2026-05-17)

## 1. 문제 해결 사항 (크론 작업 및 API 오류 수정)

블로그 자동 발행 크론(cron) 작업이 정상적으로 동작하지 않거나 오류가 발생하는 문제를 다음과 같이 해결했습니다.

### 1) 크론 타임존 동기화
*   **문제**: 시스템 타임존이 KST로 변경되었으나, `cron` 데몬이 재시작되지 않아 기존 타임존(UTC)을 기준으로 예약 시간이 동작하고 있었습니다. (예: 아침 9시 작업이 저녁 6시에 실행됨)
*   **해결**: `sudo systemctl restart cron` 명령을 통해 `cron` 데몬을 재시작하여 정상적으로 한국 시간(KST)에 맞춰 실행되도록 수정했습니다.

### 2) GWS CLI 환경 변수(PATH) 누락 오류 수정
*   **문제**: 크론 작업으로 백그라운드에서 스크립트 실행 시, `gws` (Google Workspace CLI)가 설치된 npm 전역 경로(`~/.local/share/fnm/...`)를 찾지 못해 블로그 발행 토큰을 발급받지 못하는 문제가 있었습니다.
*   **해결**: `core/blogger_api.py` 내부에서 `gws` 명령어를 호출할 때, 강제로 해당 설치 경로를 `PATH` 환경 변수에 추가하도록 수정했습니다.

### 3) 로컬 AI 에이전트(Claude, Gemini) 환경 변수 로드
*   **문제**: 크론이 기본 쉘(Shell)로 실행될 때, 사용자의 `.bashrc` 환경 변수(API 키 및 로컬 AI CLI 도구들)를 불러오지 못하여 템플릿 모드(Fallback)로 동작했습니다.
*   **해결**: `web/api_server.py`에서 크론 작업을 등록할 때 `/bin/bash -l -c` (로그인 쉘)를 거치도록 수정하여, 크론 실행 시에도 사용자의 모든 환경 변수가 정상적으로 로드되도록 조치했습니다.


## 2. 프론트엔드(대시보드) 실행 방법

웹 브라우저를 통해 파이프라인 관리, 단발성 포스팅, 실시간 로그 확인 등을 할 수 있는 **Auto-Blogger Pro 대시보드** 실행 방법입니다.

### 실행 방법
터미널에서 `auto-blogger/web` 디렉터리로 이동한 뒤, `api_server.py`를 실행합니다.

```bash
cd /home/ubuntu/auto-blogger/web
python3 api_server.py
```
*(백그라운드에서 실행하고 싶다면 `python3 api_server.py > server.log 2>&1 &` 명령어를 사용하세요.)*

### 접속 주소
서버가 실행된 후, 브라우저에서 아래 주소로 접속합니다.
👉 **http://localhost:8000/**
